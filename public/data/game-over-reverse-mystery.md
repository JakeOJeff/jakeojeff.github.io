# a murder mystery told BACKWARDS - game over

## Date : 08-09-2025

made a game called Game Over and the whole hook is one question, what would a murder mystery feel like if it played in reverse? instead of starting with a body and working forward to the killer, you start at the end and rewind through the events. its a puzzle game where you unwind a murder instead of solving it

i love a good structural gimmick and this one really grabbed me. telling a story backwards forces you to think about cause and effect completely differently, because the player already knows the outcome and has to reconstruct HOW it happened. every clue has to make sense both forward and backward, which was a genuine headache to design but so satisfying when a piece clicked into place. its built in lua and love2d and its more of a tight narrative puzzle than a big game

this is the kind of project where the IDEA does most of the heavy lifting. the mechanics are relatively simple, but the reverse framing makes the whole thing feel fresh. i put it up on itch with builds for windows and the .love file for everyone else. i think my favorite games to make are the ones that start from a single "what if you told it THIS way" instead of "what if there was a game about X"

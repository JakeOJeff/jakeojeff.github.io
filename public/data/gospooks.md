# trying go with a horror cli game - gospooks

## Date : 29-10-2025

got the itch to learn a new language so i made GoSpooks, a horror escape game that runs entirely in the terminal, built in go. figured the best way to learn go isnt reading docs, its building something dumb and fun with it. and a spooky text-based escape game felt very on theme for late october

go is such a different vibe from lua. its strict where lua is loose, it compiles down to a single static binary with zero dependencies which is honestly beautiful, you just hand someone one file and it runs. no runtime, no "install these 40 packages" nonsense. and the whole language kind of pushes you toward simple, boring, readable code, which after years of löve spaghetti was weirdly refreshing

making a game with NO graphics, just text and terminal output, forced me to think about atmosphere completely differently. all the horror has to come from writing and pacing and what you choose to reveal, theres no jumpscare art to lean on. it was a fun constraint. go's definitely joining my toolkit for backend and cli stuff, i can already tell im gonna reach for it a lot
